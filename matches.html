技术说明 · How Real-Time Works

实时更新能力说明

当前版本为高精度静态模拟。以下是三种升级方案，实现真正的每日自动更新

升级路线图 — 实时更新方案

Level 1 · 当前版本（已完成）
手动更新 js/matches-data.js 文件即可刷新全部内容。每天只需 5 分钟维护，适合个人使用。
Level 2 · 半自动（免费 API）
接入 API-Football（每日 100 次免费调用）或 TheSportsDB（完全免费）。用 Node.js 脚本每天拉取阵容、赛果数据，自动更新 JSON 文件。
Level 3 · 全自动（带新闻）
增加 Google News RSS 爬虫（通过 CORS 代理），自动抓取伤病、更衣室新闻。结合 AI 摘要生成预测调整理由。每小时自动刷新。
裁判数据来源
FIFA 官网裁判历史数据 + RSSSF 存档 + Opta 裁判执法统计。每届世界杯执法数据可追溯到 1994 年。
需要什么才能做到？
最简方案：一个 Node.js 脚本（约 50 行）+ 免费 API Key，放在任意服务器或 GitHub Actions 每天定时执行，数据直接写回本项目。
概率动态调整机制
当检测到伤病/停赛消息时，系统自动按 Elo 影响系数调整胜率：主力前锋缺席 → 对应队胜率 -5% 至 -12%。
赛前预测基于公开数据与统计模型，伤病与传言信息来源为模拟新闻报道。实际比赛结果受众多不可预测因素影响。 数据最后更新：June 11, 2026, 09:30 ET
